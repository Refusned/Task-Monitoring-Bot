Task Monitoring Bot

Что делает бот

Полный цикл заказа — без ручной рутины

  1. Создаёт заказ.Пошаговый мастер на кнопках, без печатания.
  2. Размещает на бирже.Сам обращается к API нужной биржи, считает цену до отправки.
  3. Мониторит статус.Опрашивает биржу по расписанию, не пропускает ничего.
  4. Независимо проверяет результат.Через Яндекс.Метрику или счётчики активности — не доверяет статусу биржи.
  5. Принимает или возвращает.Хорошая работа — оплата. Слабая — возврат на доработку.
  6. Раз в неделю — отчёт.Сводка автоматически в Google Sheets, без напоминаний.

💰 Баланс

Сколько денег осталось на каждой бирже — одним тапом, в реальном времени.

Что увидите

  • Живой баланс по каждой бирже, отдельной строкой
  • Зелёная галочка — биржа отвечает, цифра свежая
  • Серая отметка — биржа не предоставляет API для баланса (например, advego). Показываем честно, без подделок
  • Если один сервис упал — балансы остальных всё равно покажутся
Кнопка

🩺 Здоровье

Состояние всех интеграций на одном экране. Подходит для ежедневной проверки «всё ли работает».

Что увидите

  • Каждая биржа — настроена или нет
  • Яндекс.Метрика и Google Sheets — подключены или нет
  • Текущий режим: LIVE (реальные деньги) или DRY RUN (тренировочный, без денег)
Бот честно показывает то что есть. Не выводит зелёную галочку, если интеграция не настроена.

📦 Новый заказ

Пошаговый мастер. Каждый шаг — выбор кнопкой, никаких сложных форм.

  1. СценарийПодписки · Лайки · Трафик из соцсетей
  2. Биржаsmmcode · prskill · unu · advego · ipgold
  3. Услуга из каталогаБот сам тянет актуальный каталог биржи, фильтрует по сценарию и показывает топ услуг с реальными ценами
  4. Целевой URLСсылка на сайт, аккаунт или пост
  5. КоличествоСколько действий заказываем
  6. ПодтверждениеКарточка-предпросмотр со всеми параметрами и итоговой ценой. Заказ уходит на биржу только после явного «Подтвердить»

📋 Заказы

Список всех активных заказов. Кликабельный — каждая строка раскрывается в детальную карточку.

Что увидите

  • Список заказов: статус, биржа, сценарий, количество, цель
  • Тап по заказу — карточка с полной информацией и кнопкой «Обновить статус»
  • «Обновить» — бот заново опрашивает биржу и показывает свежее состояние

🧾 На проверку

Очередь сабмишенов от исполнителей, по которым нужно принять решение лично.

Когда сюда что-то приходит

  • Исполнитель сдал работу на бирже микрозадач
  • Бот независимо проверил результат
  • Результат на грани — нужно человеческое решение

Две кнопки

  • ✅ Принять — оплата уходит исполнителю
  • ❌ Вернуть на доработку — деньги не тратятся, исполнитель переделывает
Двойной клик невозможен — защита на уровне базы данных.

🔎 Проверить

Принудительный опрос всех активных заказов прямо сейчас.

  • По расписанию бот опрашивает биржи автоматически каждые 5 минут
  • Если хочется свежие данные не дожидаясь — нажимаете кнопку и получаете отчёт по всем заказам
  • Удобно перед звонком клиенту или важным решением

📄 Отчёт

Еженедельная сводка по трафику из соцсетей — автоматически в Google Sheets.

Как это работает

  • Каждый понедельник в 9:00 бот собирает данные за прошлую неделю и пишет в Google Sheets, вкладка «Трафик из соц сетей»
  • Колонки: неделя · платформа-источник · биржа · заказано · фактически · стоимость · статус
  • В боте — кнопка для предпросмотра текущей недели до отправки
Кнопка

📊 Сводка

Быстрый обзор всей системы — для понимания общей картины.

  • Сколько заказов в каждом статусе
  • Сколько сабмишенов ждёт решения
  • Сколько потрачено сегодня
  • Сколько ещё можно потратить (дневной лимит)
  • Текущий режим работы — LIVE или DRY RUN
Под капотом

🛡 Деньги под защитой на трёх уровнях

  1. Лимиты на тратыЛимит на один заказ и дневной лимит. Превышение — заказ не уходит на биржу.
  2. Подтверждение админаПеред каждой денежной операцией — явная кнопка «Подтвердить». Никаких автоматических списаний.
  3. Проверка цены до отправкиБот сначала спрашивает у биржи актуальную цену по каталогу, считает итог, и только потом отправляет заказ. Если получается дороже — отказывает сразу.
Заказ не может уйти дважды, сабмишен не может быть оплачен повторно — защита на уровне базы данных, не только в логике.
