### 2. Live-сценарий через Telegram (3 минуты)
Бот: **@bot_of_task_bot**

Пишем что-то вроде:
> «Накрути 20 просмотров на https://www.youtube.com/watch?v=dQw4w9WgXcQ»

LLM на kimi-k2.6 через Ollama Cloud:
1. Парсит намерение (metric=views, platform=youtube, quantity=20)
2. `get_quote` параллельно у 5 бирж → smmcode дешевле всех (~₽5)
3. `get_balances` → smmcode единственная с деньгами
4. `capture_snapshot` → YT Data API возвращает текущее число просмотров (baseline)
5. `place_order` → реальный заказ на smmcode
6. Ответ в Telegram: «Размещено через smmcode за X ₽»

Через несколько минут scheduler сам:
- Опросит статус заказа
- Вычислит дельту просмотров через YT API
- Пушнет финальный отчёт пользователю

## Главные тезисы (если спросит)

**Q: А если LLM ошибётся / зациклится / попытается потратить лишнее?**
A: Money safety зашита в код, не в промпт. `place_order` идемпотентен по `client_order_uuid` (C1) — двойного списания не будет. `max_cost` валидируется в адаптере до вызова биржи (HIGH-c). `capture_snapshot` обязателен для `place_order` — нельзя купить, не сняв baseline.

**Q: Почему модель `kimi-k2.6`, а не Claude/GPT?**
A: На Ollama Cloud в 2026 топовый tool-calling по BFCL: `kimi-k2.6`, `glm-5.1`, `qwen3-next:80b`. Один ключ в конфиге переключает модель за 5 секунд — закладывалось как config-driven.

**Q: Что с другими 4 биржами?**
A: Адаптеры все 5 готовы, абстракция через `PanelAdapter`/`TaskExchangeAdapter` + Capability flags. Smmcode подключен полностью с реальным каталогом. Остальные требуют валидных API-ключей в `.env` — после их предоставления это конфиг-задача (5 минут), не разработка.

**Q: Верификация для других платформ?**
A: ABC `PlatformVerifier` готов, диспетчер по `(platform, metric)` готов. YouTube — реальная Data API v3. VK / Telegram / Instagram — слоты ready, постфактум подключаются как `verification/<platform>.py`.

**Q: Стек устойчив?**
A: Backend и OpenClaw — два systemd unit с `Restart=always`. Watchdog раз в минуту проверяет здоровье. Реальный uptime сегодня — устойчиво держится после конфиг-правок без падений.

**Q: Сколько уйдёт на «допилить полностью»?**
A: ~5 рабочих дней:
1. Подключить реальные API остальных 4 бирж (получить ключи + 50 строк service_id_map на каждую) — 1 день
2. Verifier для VK + Telegram (официальные API, бесплатные квоты) — 1 день
3. Реальная интеграция платежей через Telegram Payments / Stripe — 1 день
4. Полная тестовая обвязка (pytest на app/, scheduler/, MCP) — 1 день
5. Документация + видео-демо + продакшн-полировка — 1 день
